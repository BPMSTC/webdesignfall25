The Bad Example uses generic <div> tags. While they look like buttons, the browser doesn't know they are interactive. Keyboard users cannot tab to them, and screen readers will skip right over them as if they were just plain text.

The Good Example uses semantic HTML. The <nav> element tells assistive technology "this is navigation". The <a> tags are automatically added to the tab order, allowing keyboard users to navigate freely.

2. Headings & Contrast

Headings create the outline of your content. Contrast ensures it's readable.

Visually, both examples might look similar to a user with perfect vision. However, the Bad Example relies entirely on CSS for hierarchy. A screen reader user navigating by "Headings" would find nothing on the page. The low contrast also makes it unreadable for users with low vision or in bright sunlight.

Without alt text, a screen reader has no choice but to read the file name, which is often gibberish (e.g., IMG_5502.jpg). The Good Example provides a concise description of the visual content, allowing blind users to visualize the product.
